记忆设计不是让系统记更多，而是让重要信息以合适的层次留下来

好的记忆设计会让协作越来越顺；糟糕的记忆设计会让系统越来越臃肿、越来越危险。

看 Safety Boundaries
Wave 1当前优先覆盖理解 → 配置 → 首次成功闭环。
多页面每个重要主题都有独立 URL，便于分享与扩展。
分层阅读所有页面都遵循“短答案 → 解释 → 下一步”的节奏。
这页关注的是“写什么、写到哪、什么时候读、什么时候清”。

核心原则

  • 把稳定偏好和长期决策沉淀到长期记忆。
  • 把临时上下文记录到日记式记忆。
  • 不要把敏感或无关信息无差别长期保留。
