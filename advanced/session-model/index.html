Session 设计会直接影响体验、成本和可维护性

如果你把所有事都堆在一个超长会话里，通常只会越来越慢、越来越糊。

看 Memory Design
Wave 1当前优先覆盖理解 → 配置 → 首次成功闭环。
多页面每个重要主题都有独立 URL，便于分享与扩展。
分层阅读所有页面都遵循“短答案 → 解释 → 下一步”的节奏。
理解 session model，本质上是在理解“如何给协作切边界”。

为什么重要

  • 影响上下文质量与 token 成本。
  • 影响任务拆分与长期协作稳定性。
  • 影响不同人、不同渠道、不同工作流是否会互相污染。
