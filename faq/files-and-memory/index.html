关键不是看到 ENOENT 就紧张，而是判断它是否真的影响你的工作流。

常见情况

  • 某些 memory 文件还没创建。
  • 系统按约定尝试读取，但文件尚不存在。
  • 这会产生日志噪音，但不一定影响核心功能。

你该做什么

  • 先判断它是不是预期文件。
  • 再判断是否真的导致功能失败。
  • 如果只是噪音，可以后续再优化，而不是立刻当成阻断。
