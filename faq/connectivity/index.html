连不上，不一定是“整个系统坏了”

很多连接问题其实集中在 gateway 状态、访问入口、token 不一致、控制界面缓存或远程可达性配置。

回到 FAQ
Wave 1当前优先覆盖理解 → 配置 → 首次成功闭环。
多页面每个重要主题都有独立 URL，便于分享与扩展。
分层阅读所有页面都遵循“短答案 → 解释 → 下一步”的节奏。
只要分层判断，通常很快能把问题缩小。

优先怀疑什么

  • Gateway 是否真的在运行。
  • 当前 UI / 控制面拿到的 token 是否正确。
  • 是不是旧页面缓存了过期状态。
  • 远程访问路径是否与当前配置一致。
