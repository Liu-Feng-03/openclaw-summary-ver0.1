Use Cases

OpenClaw 在 GitHub 场景里最有价值的地方，是把“看 issue、做修改、提 PR、看 CI”串成可协作流程

它并不是替代开发者，而是把许多重复性的查询、检查和执行动作组织得更顺手。

典型工作流

  • 看 issue / PR / CI 状态。
  • 让代理做代码实现与修复。
  • 拆分子任务给 subagent / ACP。
  • 在本地工作区和 GitHub 之间形成闭环。
