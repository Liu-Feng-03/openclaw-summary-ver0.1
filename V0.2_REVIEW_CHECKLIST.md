---

## 1. Purpose

This checklist is used to review each new v0.2 product build.

Its role is to make review more consistent, faster, and less dependent on vague subjective memory.

Use it when:
- reviewing local builds
- comparing iterations
- deciding whether the next cycle should stay in design or return to implementation
- checking whether the product is becoming more complete rather than just more complex

---

## 2. How to use this checklist

For each area below, evaluate the build using one of these labels:

- **Pass** = clearly good enough for current stage
- **Partial** = improved but still noticeably incomplete
- **Fail** = still a major product problem
- **N/A** = not applicable to the pages being reviewed yet

Optional quick score:
- 2 = Pass
- 1 = Partial
- 0 = Fail

This checklist is not only for pixel review.
It covers:
- navigation
- orientation
- interaction
- page hierarchy
- visual system
- component consistency
- page completeness

---

## 3. Fast Review Order

When reviewing a new build, check in this order:

1. Homepage
2. Module page (Getting Started)
3. Module page (Concepts)
4. Module page (Setup)
5. Module page (FAQ)
6. One detail page from each of the above
7. Mobile/responsive spot-check
8. Long-tail / non-flagship pages

This order catches the most important system problems fastest.

---

# 4. Core Product Orientation

## 4.1 Do I know what this site is within a few seconds?
- Pass criteria:
  - homepage clearly explains product purpose quickly
  - title and support copy are easy to scan
- Red flags:
  - homepage feels like a vague marketing block
  - too much reading required before understanding the product

## 4.2 Do I know how I am supposed to use the site?
- Pass criteria:
  - homepage strongly suggests how to begin
  - route options feel understandable
- Red flags:
  - the site still feels like a directory
  - user must invent their own reading strategy from scratch

## 4.3 Does the product feel like a learning product rather than a docs mirror?
- Pass criteria:
  - guidance, previews, and next-step logic feel intentional
- Red flags:
  - pages still feel like loosely connected content buckets

---

# 5. Navigation and “Where Am I?” Review

## 5.1 Do I always know which module I am in?
- Pass criteria:
  - active module state is clear in top nav and/or explorer
- Red flags:
  - module identity is vague after entering a page

## 5.2 Do I always know which exact page I am on?
- Pass criteria:
  - breadcrumb or page-local context is obvious
- Red flags:
  - user must infer current location from body content

## 5.3 Can I see what else exists around the current page?
- Pass criteria:
  - explorer / local navigation reveals nearby structure
- Red flags:
  - page still feels isolated
  - user can only move by going back or guessing links

## 5.4 Does the product provide both map sense and route sense?
- Pass criteria:
  - explorer provides structure awareness
  - next step / prev-next provide progression
- Red flags:
  - only one type of navigation is working

## 5.5 Is the explorer / side navigation genuinely helpful?
- Pass criteria:
  - it reduces confusion
  - it shows current location clearly
  - it is not visually overwhelming
- Red flags:
  - it feels like a raw sitemap dump
  - it is present but not useful

## 5.6 On mobile, is orientation still preserved?
- Pass criteria:
  - explorer/drawer access exists
  - breadcrumb or equivalent still works
- Red flags:
  - mobile strips away too much navigation context

---

# 6. Homepage Review

## 6.1 Is the homepage title visually under control?
- Pass criteria:
  - headline is clear but not oversized or oppressive
- Red flags:
  - first impression is still “this title is too big / too loud”

## 6.2 Is the primary CTA obvious?
- Pass criteria:
  - user sees what the recommended first move is
- Red flags:
  - too many equal-weight entry points compete for attention

## 6.3 Do the three entry modes feel useful and distinct?
- Pass criteria:
  - stage / goal / module entry each feels meaningful
- Red flags:
  - they feel repetitive or conceptually overlapping

## 6.4 Does the homepage feel more like a routing page than a directory?
- Pass criteria:
  - sequence and emphasis guide decision-making
- Red flags:
  - homepage is still mostly a pile of links/cards

## 6.5 Is the homepage visually calmer than before?
- Pass criteria:
  - less pressure from oversized hero elements
  - route blocks share attention well
- Red flags:
  - hero still dominates too aggressively

---

# 7. Module Page Review

## 7.1 Does each module page clearly communicate what the module is for?
- Pass criteria:
  - a user understands the job of the module immediately
- Red flags:
  - module intro is too vague or too generic

## 7.2 Do module pages feel like knowledge hubs rather than category lists?
- Pass criteria:
  - expandable previews and route hints are working
- Red flags:
  - module pages still feel like glorified link lists

## 7.3 Does each module feel visually and structurally different enough?
- Pass criteria:
  - Getting Started, Concepts, Setup, FAQ, etc. feel like different product spaces
- Red flags:
  - they still feel like the same template with different text

## 7.4 Are expandable topic cards actually helpful before navigation?
- Pass criteria:
  - user can learn something meaningful before opening a full page
- Red flags:
  - expansion feels too shallow or redundant

## 7.5 Does each module page provide a strong “start here” route?
- Pass criteria:
  - users can confidently begin without scanning the whole page
- Red flags:
  - important starting points are visually buried

---

# 8. Detail Page Review

## 8.1 Do detail pages feel like reading pages, not mini landing pages?
- Pass criteria:
  - header is lighter and content is the main focus
- Red flags:
  - detail pages still feel over-designed at the top

## 8.2 Does the page answer “what is this / why it matters / what next” clearly?
- Pass criteria:
  - summary box and next-step logic are strong
- Red flags:
  - user finishes reading without clear direction

## 8.3 Is the quick answer / summary box genuinely useful?
- Pass criteria:
  - gives fast understanding before full reading
- Red flags:
  - summary is too vague or too redundant

## 8.4 Are related pages and prev/next navigation helping, not cluttering?
- Pass criteria:
  - they feel relevant and easy to use
- Red flags:
  - they feel mechanically attached

## 8.5 Could a shared-entry visitor understand where this page fits?
- Pass criteria:
  - page provides enough context when opened directly
- Red flags:
  - user needs prior browsing to interpret the page properly

---

# 9. Card and Component Review

## 9.1 Are card title and body spacing comfortable?
- Pass criteria:
  - cards feel breathable
- Red flags:
  - title and summary feel cramped

## 9.2 Do same-row cards align cleanly?
- Pass criteria:
  - no obviously messy height mismatch in equal-height groups
- Red flags:
  - the row looks uneven and untidy

## 9.3 Are card sizes and internal rhythm consistent?
- Pass criteria:
  - cards in the same family look like siblings
- Red flags:
  - each card feels slightly improvised

## 9.4 Are CTAs placed consistently?
- Pass criteria:
  - action placement feels predictable
- Red flags:
  - buttons jump around or float inconsistently

## 9.5 Do components feel semantically different where they should?
- Pass criteria:
  - route cards, topic cards, accordion items, next-step blocks feel meaningfully distinct
- Red flags:
  - everything still feels like the same card with minor styling changes

---

# 10. Visual System Review

## 10.1 Is typography hierarchy calm and role-appropriate?
- Pass criteria:
  - homepage, module, and detail titles feel intentionally tiered
- Red flags:
  - inner pages still use headline treatments that feel too large or too loud

## 10.2 Are sections visually distinct in weight and spacing?
- Pass criteria:
  - major vs support vs utility sections are easy to distinguish
- Red flags:
  - all sections feel equally heavy

## 10.3 Does the interface feel cleaner and more aligned than before?
- Pass criteria:
  - grids, margins, spacing, and card rhythms feel stable
- Red flags:
  - visual small messes keep accumulating

## 10.4 Does each module have recognizable identity?
- Pass criteria:
  - some visual cues clearly differ per module
- Red flags:
  - module identity still depends almost entirely on text labels

## 10.5 Has repetitive visual scaffolding been reduced enough?
- Pass criteria:
  - repeated hero/KPI/template feel is gone
- Red flags:
  - pages still feel copy-pasted at the top

---

# 11. Interaction Review

## 11.1 Do interactions reduce cognitive load?
- Pass criteria:
  - expansion, navigation, and quick paths make choosing easier
- Red flags:
  - interactions look nice but do not really help understanding

## 11.2 Do module-page expansions reduce jump anxiety?
- Pass criteria:
  - user can preview confidently before leaving the page
- Red flags:
  - user still feels forced into blind navigation

## 11.3 Does FAQ interaction feel like triage rather than browsing?
- Pass criteria:
  - symptom/category flow is strong
- Red flags:
  - FAQ is still basically a page list

## 11.4 Does Setup interaction feel like a task flow?
- Pass criteria:
  - progress, checkpoints, and sequence are visible
- Red flags:
  - Setup still reads like normal content rather than guided action

## 11.5 Does Concepts interaction help reveal relationships?
- Pass criteria:
  - the module gives some sense of structure, grouping, or connections
- Red flags:
  - Concepts is still mostly a flat set of cards

---

# 12. Content System Review

## 12.1 Do flagship pages feel strong enough?
- Pass criteria:
  - core pages are clearly more complete and useful
- Red flags:
  - important pages still feel thin or generic

## 12.2 Do non-flagship pages still feel inside the same system?
- Pass criteria:
  - even lighter pages feel intentionally integrated
- Red flags:
  - long-tail pages feel leftover or placeholder-like

## 12.3 Is there still a quality cliff between strong pages and weak pages?
- Pass criteria:
  - no dramatic drop in coherence across the site
- Red flags:
  - only a handful of pages feel “real” and the rest feel unfinished

## 12.4 Are page completion levels being handled honestly?
- Pass criteria:
  - lighter pages are still structured and useful
- Red flags:
  - incomplete pages are disguised rather than systematized

---

# 13. Final Product Feeling Review

## 13.1 Does the site now feel like a real product?
- Pass criteria:
  - coherent, navigable, deliberate, trustworthy
- Red flags:
  - still feels like a polished prototype rather than a product

## 13.2 Could I confidently share this with a new user?
- Pass criteria:
  - I know where I would send them and trust their first experience
- Red flags:
  - I still need to explain how to use the site before sharing it

## 13.3 Is the product getting simpler to understand as it grows?
- Pass criteria:
  - expansion in scope does not create proportionate confusion
- Red flags:
  - more pages just means more complexity

## 15. Short Handoff Summary

If another process reads only this file, the key message is:

> Do not review v0.2 by asking only “does it look better?”

Review it by asking:
- Do I know where I am?
- Do I know how to move?
- Do pages feel different when they should?
- Are cards and layout disciplined?
- Is the whole site becoming more complete, not just prettier?
