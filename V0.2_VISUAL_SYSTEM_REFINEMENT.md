# V0.2_VISUAL_SYSTEM_REFINEMENT.md — openclaw-summary

Last updated: 2026-03-31
Status: visual-system refinement spec for v0.2

---

## 1. Why this document exists

After the first v0.2 implementation direction, the product is already much better than v0.1.

However, several refinement issues still reduce product completion feel:
- homepage title is still too large / too dominant
- cards feel too tight internally in some places
- same-row cards do not always align cleanly
- some page groups still feel visually uneven or messy
- module differentiation exists conceptually, but not strongly enough visually

This document defines the next-layer visual refinement rules.

---

## 2. Visual refinement goals

The visual system should now optimize for:
- calmer hierarchy
- stronger alignment discipline
- cleaner card rhythm
- more obvious page-type differences
- more obvious module identity
- less template repetition
- better finished-product feel

---

## 3. Primary visual diagnosis

### D1. Titles are still too aggressive in some contexts
Especially homepage and top blocks can still feel oversized and visually loud.

### D2. Card internal spacing is not fully stable
Title, summary, metadata, and CTA zones are sometimes too close.

### D3. Card groups are not always visually aligned
Mixed heights and uneven spacing make some sections feel untidy.

### D4. Modules need stronger visual identity
They still risk feeling like the same page system wearing slightly different labels.

### D5. Visual rhythm across blocks needs stronger rules
Some sections feel too similar in weight or spacing.

---

## 4. Page-Type Visual Hierarchy

The product must use three clearly different visual hierarchy modes.

### 4.1 Homepage mode
Character:
- calm but welcoming
- route-oriented
- slightly more expressive than inner pages

Rules:
- headline should be shorter and tighter than current draft
- primary CTA should appear early
- avoid oversized text block dominating the whole hero
- visual emphasis should go to user action, not self-description

### 4.2 Module page mode
Character:
- compact
- structured
- utility-forward

Rules:
- no oversized landing-page hero
- header should feel like a module panel, not a marketing banner
- left explorer + module heading should visually dominate over decorative header treatment

### 4.3 Detail page mode
Character:
- reading-oriented
- focused
- low-friction

Rules:
- headers should be lighter and shorter than module headers
- typography should encourage reading, not impressiveness
- visual emphasis should shift to content sections and related navigation

---

## 5. Homepage Header Refinement

### 5.1 Problem
The homepage headline is still too large and too visually forceful.

### 5.2 New rule
Homepage title should feel like:
- a clear orientation line
- not a giant campaign slogan

### 5.3 Recommended structure
- eyebrow / small label
- headline in 1–2 lines max
- one support sentence
- CTA row immediately after

### 5.4 Recommended design constraint
- reduce font size compared with current implementation
- reduce line-height pressure
- reduce top padding if necessary
- headline should not be the largest emotional object on the page; the route blocks should share attention

---

## 6. Header Size Rules by Page Type

### Homepage
- largest title in product, but still controlled
- max 1–2 lines preferred

### Module page
- medium title
- shorter and more functional
- not dramatically larger than section headings

### Detail page
- smaller than module page title
- optimized for reading context

### Core principle
A user should immediately feel page depth from title treatment alone.

---

## 7. Card Layout System

Cards are now a major part of the product, so they need stricter rules.

### 7.1 Standard card internal structure
1. title
2. summary
3. metadata row (if needed)
4. secondary explanatory content or expansion cue
5. CTA zone

### 7.2 Spacing rules
Must define stable vertical rhythm between:
- title and summary
- summary and metadata
- metadata and CTA
- content and card boundary

### 7.3 Problem to avoid
Do not let cards become visually cramped because title and body are too close.

### 7.4 Recommended rule
Every card type should have a minimum internal spacing system rather than local ad hoc adjustments.

---

## 8. Card Height and Alignment Strategy

### 8.1 Problem
Cards in the same row can feel messy when heights differ too much.

### 8.2 Rule
For the same card group, use a predictable alignment strategy.

### Options
#### Option A — Equal-height rows
Best for:
- homepage route cards
- module exploration cards
- shareable link cards

#### Option B — Natural-height cards with stricter rhythm
Best for:
- long expandable cards
- FAQ accordion content
- detail-page related cards with variable text

### 8.3 Recommendation
Use equal-height cards where direct side-by-side comparison matters.
Use natural height only where expansion or content variability is part of the design.

### 8.4 CTA alignment rule
For equal-height cards, CTA area should align toward the bottom where practical.
This dramatically improves visual neatness.

---

## 9. Title and Summary Truncation Rules

### 9.1 Why needed
Uncontrolled text length causes uneven card height and messy rhythm.

### 9.2 Recommended rule
- card titles: prefer consistent line clamp rules
- summaries: clamp to a stable number of lines in equal-height groups

### 9.3 Exception
Expanded cards can reveal full content later.

---

## 10. Section Rhythm System

### 10.1 Problem
Different sections can feel too similar in weight and spacing.

### 10.2 Rule
Define consistent section rhythm:
- section header spacing
- section-to-section vertical gap
- inner group spacing
- block emphasis tiers

### 10.3 Three section weight levels

#### Tier 1 — Major sections
Examples:
- homepage route selection
- guided learning path
- module expandable content

#### Tier 2 — Support sections
Examples:
- quick path blocks
- related module bridge
- shareable links

#### Tier 3 — Utility sections
Examples:
- metadata strips
- footer utilities
- small contextual notes

Each tier should feel different in spacing and visual prominence.

---

## 11. Module Visual Identity System

### 11.1 Current issue
Modules are conceptually different but not yet visually distinct enough.

### 11.2 Goal
A user should be able to feel:
- this is onboarding
- this is concepts
- this is setup
- this is FAQ

without reading every word.

### 11.3 Identity levers
Use some combination of:
- accent color
- iconography
- header structure
- card shape nuance
- section motif
- interaction style emphasis

### 11.4 Suggested module identity directions

#### Getting Started
- warm / welcoming / confidence-building
- path and onboarding motifs

#### Concepts
- structure / connection / node / relation motifs

#### Setup
- checklist / progress / verification motifs

#### FAQ
- diagnosis / accordion / symptom triage motifs

#### Use Cases
- scenario / comparison / value demonstration motifs

#### Advanced
- depth / systems / expertise motifs

### 11.5 Important limit
Do not make modules visually theatrical.
The system should remain coherent and professional.

---

## 12. Decorative Element Discipline

### 12.1 Problem
Repeated generic decorative patterns can quickly look templated.

### 12.2 Rule
Decoration must follow function.

Examples:
- relation motifs belong more in Concepts
- progress/checkpoint motifs belong more in Setup
- triage and category emphasis belong more in FAQ

### 12.3 Avoid
- repeating the same badge cluster everywhere
- repeating the same hero side panel everywhere
- decorative gradients doing the same thing on every page

---

## 13. Typography Refinement

### 13.1 Goals
- calmer hierarchy
- better scanability
- less pressure from oversized titles
- clearer relationship between title, summary, metadata, and section body

### 13.2 Recommended typography zones
- display / homepage title
- module heading
- detail heading
- section heading
- card title
- body summary
- metadata text

Each should have a stable role and not drift too close to another level.

### 13.3 Card title rule
Card titles must feel readable and structured, not like mini hero headlines.

### 13.4 Metadata rule
Metadata should be quieter than summary text, never competing with title.

---

## 14. Grid and Alignment Rules

### 14.1 Problem
Visual neatness drops when card groups are not aligned consistently.

### 14.2 Rules
- cards within a comparison group should align on top edges
- equal-height groups should align bottoms where CTA exists
- section headers should align consistently with main content grid
- explorer/content/toc relationship should be structurally intentional

### 14.3 Recommended layout emphasis
On desktop inner pages, the strongest structural alignment should be:
- explorer column
- content column
- optional secondary utility column only if needed

Not:
- oversized decorative hero + floating random card clusters

---

## 15. Visual Cleanliness Checklist

Before calling a page visually complete, verify:
- title is not oversized for its page type
- card groups feel aligned
- cards do not feel cramped internally
- same-level items feel like the same family
- sections have readable spacing contrast
- module identity is visible but not loud
- visual decoration supports function

## 16. Relationship to Navigation System

This document should be used together with `V0.2_NAVIGATION_SYSTEM.md`.

Important interaction:
- once explorer is added, page headers should become even lighter
- orientation should come more from structure and alignment, less from giant visual headers

This is a major shift toward product maturity.

## 17. Short Handoff Summary

If another process reads only this file, the main instruction is:

> The next refinement stage is not about adding more decoration.
> It is about reducing visual noise, calming hierarchy, stabilizing card rhythm, and making modules feel intentionally different.

The highest-priority visual improvements are:
- smaller and calmer homepage title
- stronger card spacing rules
- cleaner same-row alignment
- stronger module identity
- lighter, more product-like headers across inner pages
