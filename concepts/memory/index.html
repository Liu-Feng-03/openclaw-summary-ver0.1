Core Concepts

OpenClaw 的“记忆”更接近文件化连续性，而不是神秘的永久脑内记忆

它强调把重要信息写进文件、分层保存，再在需要时检索。

看 Setup看 Advanced: Memory Design
Wave 1当前优先覆盖理解 → 配置 → 首次成功闭环。
多页面每个重要主题都有独立 URL，便于分享与扩展。
分层阅读所有页面都遵循“短答案 → 解释 → 下一步”的节奏。
这比幻想模型天然长期记住一切更可靠，也更适合真实协作。

要先建立的正确直觉

  • 会话上下文是短期的，会变长、会切换、也会被重置。
  • 真正值得保留的信息，应写入 memory/YYYY-MM-DD.md 或 MEMORY.md 之类的文件。
  • 记忆不是越多越好，而是越有结构越好。

为什么文件记忆很重要

  • 它能跨 session 保持连续性。
  • 它更容易审计、编辑、纠正和清理。
  • 它天然支持安全边界：什么能长期保留，什么不该泄露。

常见边界

Daily memory

适合记录最近发生的事与临时上下文。

Long-term memory

适合保留稳定偏好、长期决策与重要背景。

不要做的事

别把所有细枝末节都塞进去，最后只会变成垃圾堆。
