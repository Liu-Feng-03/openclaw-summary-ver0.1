Core Concepts

Session 是 OpenClaw 里最重要的上下文边界之一

你可以把它理解成“某一段持续协作的工作容器”，里面会积累对话、工具使用痕迹、任务上下文和状态。

看 Gateway看 Memory
Wave 1当前优先覆盖理解 → 配置 → 首次成功闭环。
多页面每个重要主题都有独立 URL，便于分享与扩展。
分层阅读所有页面都遵循“短答案 → 解释 → 下一步”的节奏。
很多“为什么它这样回答”“为什么上下文变多了”“为什么要开新线程”之类的问题，都和 session 有关。

Session 是什么

  • 它不是单纯的聊天窗口，而是一次持续协作的上下文单元。
  • 它决定模型当前能看到哪些历史信息，也影响后续动作和成本。

为什么 session 边界很重要

  • 上下文越长，不代表越聪明；很多时候只会更重、更慢、更乱。
  • 产品讨论、代码实现、复杂诊断拆到不同 session，通常更稳定。
  • 不同人、不同渠道、不同任务也应有不同边界，避免信息串味。

常见误解

误解：开新 session 会“失忆”

实际上重要内容应该写入文件记忆，而不是强行塞在上下文里。

误解：所有事都堆一个 session 最方便

短期省事，长期通常更糟。
